# openCrawler 需求規劃文件

版本:1.0 · 建立日期:2026-04-23

---

## 1. 專案目的

提供一款**跨平台桌面應用程式**(Windows + macOS),讓使用者能一鍵把網頁文章保存成**可離線閱讀**的本地副本,依**資料夾分類**管理,並可將單篇或整個資料夾送入 **Google NotebookLM Enterprise** 產生摘要或進行延伸分析。

定位:個人知識庫前置蒐集器 + NotebookLM 資料來源管理器。

---

## 2. 目標使用者與使用情境

**目標使用者**:研究人員、產品經理、設計師、工程師等需要長期收藏網頁資料並做內容整理的人。

**典型情境**:

| 情境 | 流程 |
|------|------|
| 日常收藏 | 看到好文章 → 貼 URL → 選分類 → 按下載 → 離線保存 |
| 閱讀複習 | 開啟 App → 點分類 → 點文章 → 在內嵌/外部瀏覽器閱讀 |
| AI 摘要 | 右鍵單篇或整個資料夾 → 「送 NotebookLM 摘要」→ 輸入 prompt → 取回結果 |
| 多裝置搬移 | 所有資料放在使用者指定的硬碟資料夾,可直接複製到其他機器 |

## 3. 功能需求(Functional Requirements)

### FR-1 首次啟動設定
- **FR-1.1** 第一次啟動時出現設定精靈,強制使用者選擇**文章儲存根資料夾**(例如 `D:\Knowledge` 或 `/Users/me/Knowledge`)。
- **FR-1.2** 使用者可在設定頁之後隨時修改儲存路徑(搬移既有檔案為**手動**,系統只更新 config)。
- **FR-1.3** 設定頁可選填 Google Cloud Service Account JSON 金鑰路徑與 Project Number、Location,供 NotebookLM API 使用。
- **FR-1.4** 設定檔儲存於平台慣例位置:
  - Windows: `%APPDATA%\openCrawler\config.json`
  - macOS: `~/Library/Application Support/openCrawler/config.json`

### FR-2 資料夾(分類)管理
- **FR-2.1** 可建立、重新命名、刪除分類資料夾。
- **FR-2.2** 分類可巢狀(支援子資料夾,最多 3 層)。
- **FR-2.3** 分類實際對應到磁碟上的資料夾結構,同時登錄於 SQLite 資料庫(作為索引、附加元資料)。
- **FR-2.4** 刪除分類時提示使用者:僅刪 DB 紀錄 / 連同檔案一併刪除 / 取消。

### FR-3 下載網頁文章
- **FR-3.1** 主畫面有一個 URL 輸入框與「下載」按鈕;下載按鈕旁有**下載模式**選項(預設「自動」/ 可選「快速」/ 可選「瀏覽器模式」)。
- **FR-3.2** 下載前需選擇目標分類(預設使用者上次選的分類)。
- **FR-3.3** **雙層下載策略**:
  - **快速模式(HttpClient + AngleSharp)** — 預設第一層。低延遲、輕量。適用大多數公開網頁。
  - **瀏覽器模式(Playwright + Chromium)** — fallback。啟動無頭瀏覽器載入頁面,等待 JS 渲染完成後抓取 DOM;可繞過:
    - Cloudflare / Akamai 簡易挑戰
    - 需要 JS 渲染才有內容的 SPA
    - 擋純 HTTP 請求的反爬蟲(UA、TLS fingerprint 檢查)
  - **自動模式(預設)**:先試快速模式,若發生以下情況自動切瀏覽器模式並重試一次:
    - HTTP 403 / 429 / 503
    - 回應 body < 500 bytes 或偵測到「Just a moment...」「Enable JavaScript」等典型挑戰字串
    - `<body>` 為空或文字內容 < 200 字元
- **FR-3.4** 下載流程(兩模式共用後半段):
  1. 取得完整 HTML(快速模式從 HttpClient;瀏覽器模式從 Playwright `page.content()`)。
  2. 解析 `<title>` 作為文章標題。
  3. 抓取 `<img src>`、`<img srcset>`、`<source srcset>`、`<link rel="stylesheet">` 及 CSS `url(...)` 圖片(瀏覽器模式下 `src` 可能已被 JS 改寫為絕對 URL,對下載更有利)。
  4. **不處理** `<video>`、`<audio>`、`<iframe>`、`<object>`、`<embed>`。
  5. **剝離 JavaScript**:移除所有 `<script>` 元素、`on*` 事件屬性、`javascript:` URL。
  6. 改寫 HTML 內的資源路徑為**相對路徑**。
  7. 寫入 `index.html` 到 `<分類資料夾>/<folder_name>/`。
- **FR-3.5** 下載過程顯示 loading 狀態,瀏覽器模式會明確標註(例:「瀏覽器模式載入中...(首次需下載瀏覽器,約 150 MB)」)。
- **FR-3.6** `meta.json` 記錄實際使用的 fetch 模式(`fetch_mode: "fast"` 或 `"browser"`),便於日後診斷。
- **FR-3.7** Playwright **瀏覽器 binary 延遲安裝**:首次使用瀏覽器模式時才提示使用者下載,而不是 App 安裝時就強制抓。使用者若拒絕,瀏覽器模式永久不可用(快速模式正常)。
- **FR-3.8** 下載完成後 SQLite 新增一筆 article 紀錄,並將文章加入左側文章清單。
- **FR-3.9** 支援使用者**修改標題**(只改顯示名稱 + DB,**不重新命名資料夾**;資料夾名以時間戳 + hash 為主)。
- **FR-3.10** 下載失敗時顯示錯誤訊息與重試按鈕;若自動模式已嘗試兩種模式仍失敗,錯誤訊息需分別列出兩次失敗原因。
- **FR-3.11** **每次下載為獨立快照**:同一 URL 再次下載會建立新的 article 與新的資料夾,不覆蓋舊資料。v1 不提供「更新快照」功能。

### FR-4 閱讀文章
- **FR-4.1** 雙擊文章或按「開啟」按鈕 → **以系統預設瀏覽器開啟本地 `index.html`**(對應架構決議 D1)。
  - v1 **不**提供內嵌 WebView。`AppConfig.UseEmbeddedBrowser` 欄位保留以利 v1.1 擴充,目前固定為 `false`。
- **FR-4.2** 使用者可在文章詳情頁看到:原始 URL、下載時間、所在分類、檔案大小、圖片數。
- **FR-4.3** 提供「在檔案總管/Finder 中顯示」按鈕。

### FR-5 NotebookLM 摘要
- **FR-5.1** 使用者可對**單篇文章**或**整個分類資料夾**觸發「送 NotebookLM 摘要」。
- **FR-5.2** 彈出對話框:
  - 顯示即將送出的文章清單。
  - 可編輯 `episodeFocus`(自訂 prompt,給 NotebookLM 用於音訊摘要)。
  - 勾選「同時產生純文字摘要(使用 Gemini API)」。
- **FR-5.3** 執行流程:
  1. 若尚未建立對應的 Notebook,先呼叫 `POST /notebooks` 建立,標題用分類名或文章名。
  2. 將文章以 `textContent` 或 `webContent` 形式呼叫 `sources:batchCreate` 加入 notebook。
  3. 呼叫 `audioOverviews` 產生音訊摘要(帶 `episodeFocus` 自訂 prompt)。
  4. 若勾選 Gemini 純文字摘要,另外呼叫 Gemini `generateContent` API,帶使用者 prompt + 文章內文。
- **FR-5.4** 產出結果(音訊連結、文字摘要)儲存於 SQLite,並在 App 內可重複查看。
- **FR-5.5** 提供「在 NotebookLM 網頁打開」按鈕(`https://notebooklm.cloud.google/notebook/<id>`)。

### FR-6 介面
- **FR-6.1** 介面語言支援 **繁體中文(zh-TW)** 與 **英文(en)** 兩種,使用者可在設定頁即時切換(切換後全 UI 立即更新,不需重啟)。
  - 首次啟動時依系統 locale 自動挑選(`zh-*` → 繁中,其餘 → 英文)。
  - 使用者選擇儲存於 `AppConfig.UiLanguage`。
  - 所有面向使用者的字串(標籤、按鈕、訊息、錯誤、對話框、tooltip)皆**不得硬編碼**,一律走資源字典,每個字串 key 需有兩種語言版本,缺漏時 fallback 至英文並寫 warning log。
  - 日期 / 數字格式跟隨當前 UI 語言(`CultureInfo.CurrentUICulture`)。
- **FR-6.2** 主畫面三欄佈局:
  ```
  ┌──────────┬──────────────┬─────────────────┐
  │ 分類樹   │ 文章清單     │ 文章詳情/操作   │
  │          │              │                 │
  │ ▸ 新聞   │ - 標題 A     │ 原始 URL        │
  │ ▸ 技術   │ - 標題 B     │ 下載時間        │
  │          │              │ [開啟] [摘要]   │
  └──────────┴──────────────┴─────────────────┘
  ```
- **FR-6.3** 頂部工具列:URL 輸入框 + 分類下拉 + 下載按鈕。
- **FR-6.4** 支援亮/暗主題(Avalonia Fluent Theme)。

### FR-7 命令列介面(CLI)與自動化測試
- **FR-7.1** 提供 CLI 子指令 `download`,讓使用者/測試在**不開啟 GUI** 的情況下下載文章:
  ```bash
  opencrawler download --url <URL> --category <CATEGORY_FOLDER_NAME>
                       [--storage <PATH>] [--mode auto|fast|browser]
  ```
  - `--url`(必填):目標網頁網址。
  - `--category`(必填):分類資料夾名;不存在則自動建立(僅建立單層)。
  - `--storage`(選填):覆寫 `config.json` 中的儲存根目錄,便於測試隔離。
  - `--mode`(選填,預設 `auto`):fetch 模式;`fast` 僅 HttpClient,`browser` 強制 Playwright,`auto` 依偵測自動切換(見 FR-3.3)。
- **FR-7.2** 執行成功時以 exit code `0` 結束,並輸出 JSON 到 stdout:
  ```json
  { "articleId": 42, "title": "...", "folderPath": "...", "imageCount": 12 }
  ```
  失敗時 exit code 非 0,錯誤訊息寫 stderr。
- **FR-7.3** CLI 走與 GUI 完全**相同**的 `Core` 服務層(`IArticleService.DownloadAsync`),確保行為一致。
- **FR-7.4** 整合測試(xUnit)**必備案例**:
  - **IT-01** 以 CLI 指令觸發下載,驗證:
    1. 指定 URL 能正確產生本地 `index.html`、`assets/`、`content.txt`、`meta.json`
    2. 指定的分類資料夾若不存在會自動建立
    3. SQLite 中 `articles` 表新增一筆,欄位內容正確
    4. 退出碼為 0、stdout 為合法 JSON
  - **IT-02** 同樣 URL 第二次下載進同一分類,視為**新文章**不覆蓋舊檔(folder_name 以時間戳+hash 唯一)。
  - **IT-03** URL 為 404 或逾時時,退出碼非 0、stderr 有明確訊息、DB 不新增。
  - **IT-04**(browser 模式)`--mode browser` 指令能成功抓一個由 WireMock 回應「需 JS 才有內容」的測試頁,最終 `index.html` 含 JS 渲染後的文字。此測試標記 `[Trait("Category", "Playwright")]`,CI 可選跑。
  - 測試使用 `WireMock.Net` 本機 HTTP 伺服器模擬,含 HTML + 多張內嵌圖片回應,不依賴外網。

### FR-8 文章註解(Markdown)
- **FR-8.1** 每篇文章可有**一份或多份** Markdown 註解文件(1:N)。預設建立第一份「預設註解」(title: `預設註解` / `Default Note`),使用者可持續追加內容、用 Markdown 標題自行分段,做成時間軸式研究筆記。
- **FR-8.1.1 單份軟上限** 每份 note 內容軟上限 **2 MB**;接近上限時 UI 主動提醒:
  - **≥ 80%(約 1.6 MB)**:狀態列顯示黃色警示「目前 1.6 MB / 2 MB,建議新增第二份 note」。
  - **≥ 95%**:彈 toast 提醒「即將達到單份上限,建議按『新增 Note』另起一份」。
  - **≥ 100%**:仍允許儲存但狀態列轉紅,並在自動儲存完成時顯示「已超過 2 MB 建議上限,效能可能下降」。不強制阻擋,給使用者自行決定。
- **FR-8.1.2 多份管理** 文章詳情頁「註解」分頁頂端有 **Note 切換列**(類似瀏覽器分頁),可:
  - 新增 note(按 `+`,輸入標題)
  - 重新命名 note(雙擊分頁標題)
  - 刪除 note(右鍵 → 刪除,會二次確認;若僅剩一份則禁刪只能清空)
  - 重新排序(拖曳分頁)
  - 每份 note 獨立計算字元數、獨立顯示儲存狀態
- **FR-8.2** 文章詳情頁提供「註解」分頁,內含 **雙欄 Markdown 編輯器**:
  - 左欄編輯(AvaloniaEdit,Markdown 語法上色)
  - 右欄即時預覽(Markdown.Avalonia)
  - 可切換「僅編輯」「僅預覽」「同步捲動」三種模式
- **FR-8.3** **自動儲存**:停止輸入 2 秒後或失去焦點時自動寫入 SQLite;頂部顯示狀態(「未儲存的變更」/「已儲存 · HH:mm:ss」)。
- **FR-8.4** 註解內容**只存 SQLite**(表 `article_notes`),文章資料夾內**不**產生 `.md` 檔(避免與外部 Markdown 同步衝突);若使用者需要外部檔,可透過「匯出為 notes.md」按鈕手動產出。
- **FR-8.5** 支援 GFM(GitHub Flavored Markdown)常見語法:標題、粗/斜體、列表、連結、圖片、引言、程式碼區塊、表格、待辦清單。
- **FR-8.6** 註解中可嵌入本地圖片:
  - 拖曳圖片到編輯器 → 複製到該文章的 `assets/notes/` 資料夾 → 插入 `![](assets/notes/xxx.png)` 相對路徑
  - 預覽區能正確顯示
- **FR-8.7** 文章列表項目若**任一 note** 有非空內容,顯示小圖示 + note 數量角標(例:📝²)。可依「有註解 / 無註解」篩選。
- **FR-8.8** 刪除文章時同時刪除所有相關 notes(`CASCADE`)。
- **FR-8.9** NotebookLM 摘要對話框提供「**一併送註解內容**」勾選(預設勾選);勾選時**每份非空 note** 會以獨立 `textContent` source 上傳,`sourceName = "{文章標題} - {note 標題}"`,可在對話框裡勾選要送哪些 notes(全選/全不選)。

## 4. 非功能需求(Non-Functional Requirements)

| 類別 | 需求 |
|------|------|
| 平台 | Windows 10+ 與 macOS 12+,x64 與 arm64 |
| 執行期 | .NET 9.0 |
| UI | Avalonia 11.2(MVVM + CommunityToolkit.Mvvm) |
| 效能 | 下載單篇中型網頁(50 張圖)應在 30 秒內完成 |
| 儲存 | 使用者資料完全放在其指定的資料夾,App 目錄本身不存文章 |
| 隱私 | 除了 NotebookLM/Gemini API,絕不向外部傳送文章內容 |
| 可攜性 | 使用者複製整個儲存資料夾到另一台電腦後,重新設定 config 指向該資料夾即可繼續使用(SQLite 檔也放資料夾根目錄) |
| 安全 | Service Account JSON 僅存本機,不上傳;絕不寫入日誌 |

## 5. 資料模型(SQLite)

資料庫檔位置:`<儲存根資料夾>/opencrawler.db`

### 資料表

**categories**
| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INTEGER PK | |
| parent_id | INTEGER FK → categories.id | NULL 表示根分類 |
| name | TEXT NOT NULL | 顯示名稱 |
| folder_name | TEXT NOT NULL | 實體資料夾名(安全化後) |
| created_at | TEXT(ISO8601) | |

**articles**
| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INTEGER PK | |
| category_id | INTEGER FK → categories.id | |
| title | TEXT NOT NULL | 使用者可改 |
| original_title | TEXT | 抓取時的原始 `<title>` |
| source_url | TEXT NOT NULL | |
| folder_name | TEXT NOT NULL | 相對於 category 的資料夾名(時間戳 + hash,永不改) |
| downloaded_at | TEXT(ISO8601) | |
| image_count | INTEGER | |
| size_bytes | INTEGER | |

**notebooks**(NotebookLM 對應)
| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INTEGER PK | |
| scope_type | TEXT | 'article' 或 'category' |
| scope_id | INTEGER | 對應的 article.id 或 category.id |
| remote_notebook_id | TEXT | NotebookLM API 回傳的 notebookId |
| created_at | TEXT | |

**sources**(NotebookLM source 對應)
| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INTEGER PK | |
| notebook_id | INTEGER FK → notebooks.id | |
| article_id | INTEGER FK → articles.id | |
| remote_source_id | TEXT | |

**summaries**(摘要結果)
| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INTEGER PK | |
| notebook_id | INTEGER FK → notebooks.id | |
| kind | TEXT | 'audio' 或 'text' |
| prompt | TEXT | 使用者輸入的 prompt |
| content | TEXT | 文字摘要內容或音訊 URL |
| created_at | TEXT | |

**article_notes**(每篇文章可有多份 Markdown 註解,1:N)
| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INTEGER PK | |
| article_id | INTEGER FK → articles.id | 非 UNIQUE,允許多份 |
| title | TEXT NOT NULL | note 顯示名(分頁標題);首份預設 `預設註解` / `Default Note` |
| content | TEXT NOT NULL DEFAULT '' | Markdown 原文(單份軟上限 2 MB,接近時 UI 警示) |
| sort_order | INTEGER NOT NULL DEFAULT 0 | 分頁排序(拖曳調整) |
| created_at | TEXT(ISO8601) | 建立時間 |
| updated_at | TEXT(ISO8601) | 最後編輯時間 |
| byte_size | INTEGER | content 的 UTF-8 位元組數,用來判斷是否接近 2 MB |
| char_count | INTEGER | 字元數 |

索引:`(article_id, sort_order)` 供快速列出排序後的 notes。

刪除文章時以 service 層 transaction 手動清除對應 notes(見架構文件 §7.6)。

## 7. 範圍之外(Out of Scope,v1 不做)

- 全文搜尋(未來可加 FTS5)
- 多使用者 / 雲端同步
- 行動裝置版(iOS/Android)
- 瀏覽器外掛
- PDF/EPUB 匯入(目前僅支援 HTML 網頁)
- 網站登入後才可見的內容(cookie 管理,v2 可在 Playwright 模式加上 cookie 注入)
- CAPTCHA / reCAPTCHA 人機驗證破解
- 自動排程抓取
- **內嵌 WebView 閱讀**(v1.1 規劃;v1 統一用系統瀏覽器開啟本地 `index.html`)
- **保留/執行網頁 JavaScript**(一律剝離,安全優先)
- **下載影音資源**(`<video>`、`<audio>`、`<iframe>`,v2 再評估)
- **NotebookLM Podcast API**(v1 僅用 `audioOverviews`)
- **「更新既有快照」功能**(每次下載皆為獨立新文章)

## 8. 風險與假設

| 項目 | 內容 |
|------|------|
| 風險 | NotebookLM Enterprise API 目前**無純文字摘要端點**,故另接 Gemini API 補足 |
| 風險 | 部分網站有反爬蟲(Cloudflare 簡易挑戰、SPA 純 JS 渲染),**以 Playwright 瀏覽器模式 fallback 解決**;登入牆、CAPTCHA、reCAPTCHA 等仍不支援 |
| 風險 | Playwright Chromium binary 約 150 MB,首次使用瀏覽器模式需下載,有網路限制環境下可能失敗 |
| 假設 | 使用者已擁有 Google Cloud 專案並啟用 Discovery Engine / Vertex AI API |
| 假設 | 使用者願意在 App 中設定 Service Account JSON 路徑 |
| 假設 | 使用者使用系統預設瀏覽器(Chrome / Edge / Safari / Firefox)閱讀;v1 不內嵌瀏覽器 |

## 9. 驗收準則(Acceptance Criteria)

1. 全新電腦安裝後首次啟動,會出現設定精靈,強制指定儲存資料夾。
2. 貼上任一公開網址(例如 Wikipedia 條目)並按下載,30 秒內完成,離線後仍可正常閱讀(圖片正常顯示)。
3. 建立分類、移動文章、重新命名文章,行為符合預期且重啟 App 後保留。
4. 對單篇文章執行「NotebookLM 摘要」,能在 NotebookLM 網頁看到新 notebook 與 source,且 App 內能取得音訊摘要連結。
5. 應用程式在 Windows 11 與 macOS 14 各至少執行一次完整流程無崩潰。
