# openCrawler 系統架構文件

版本:1.0 · 建立日期:2026-04-23

---

## 1. 架構原則

| 原則 | 說明 |
|------|------|
| **離線優先** | 所有文章內容皆本地保存,無網路仍可閱讀 |
| **使用者擁有資料** | 文章與 SQLite 都放在使用者指定資料夾,App 目錄只有設定檔 |
| **薄 UI / 厚 Service** | View 只做綁定,業務邏輯集中在 Service 層,方便測試 |
| **async 貫穿到底** | I/O 全走 async,UI thread 絕不被阻塞 |
| **跨平台抽象最小化** | Avalonia 已處理大部分差異,檔案路徑用 `Path.Combine`,避免寫死分隔符 |
| **可攜帶 (Portable)** | 使用者複製整個儲存資料夾即可搬移 |

## 2. 技術選型

| 層 | 選型 | 原因 |

| Runtime | .NET 9.0 | 使用者指定;LTS 選 .NET 8 或最新 .NET 9 皆可,採 9 取較新 API |

| 測試 | xUnit + NSubstitute + **WireMock.Net** | WireMock.Net 啟動本機 HTTP 模擬站,用於 CLI 端對端下載測試 |
| CLI | **System.CommandLine** 2.x (beta) | 解析 `opencrawler download --url --category`,與 GUI 共用 `Core` |

## 5. 資料流:下載一篇文章

```
使用者貼 URL 按下載
      │
      ▼
MainViewModel.DownloadCommand
      │
      ▼
IArticleService.DownloadAsync(url, categoryId)
      │
      ├─ 產生 folder_name:"yyyyMMdd_<rand4>_<slugified-title-guess>"
      ├─ 在 categoryPath 下建立資料夾 + assets/
      │
      ▼
IWebDownloader.DownloadAsync(url, folder)
      │
      ├─ GET url → AngleSharp 解析
      ├─ 抓 <title> 為 original_title
      ├─ 找 <img[src]>, <img[srcset]>, <source[srcset]>, <link[rel=stylesheet]>,
      │   CSS url(...), inline <style>
      ├─ 對每個資源:
      │    GET → 寫入 assets/<hash>.<ext>
      │    改寫原 DOM 屬性為 "assets/<hash>.<ext>"
      ├─ 寫入 index.html(改寫後)
      ├─ 寫入 content.txt(Readability 提取正文)
      ├─ 寫入 meta.json(原 url、抓取時間、header 等)
      └─ 回傳 DownloadResult { title, imageCount, sizeBytes, ... }
      │
      ▼
ArticleRepository.InsertAsync(article)  → SQLite
      │
      ▼
Event: ArticleAdded → ArticleListVM 更新
```

進度事件(`DownloadProgress`):
```csharp
public record DownloadProgress(
    string Stage,      // "FetchingHtml" | "DownloadingAssets" | "Writing"
    int Current,
    int Total,
    string? Message);
```

## 6.5 CLI 流程與整合測試

CLI 是 **GUI 的同源輕量替身**,同時也是自動化測試入口。

### 6.5.1 指令介面

```bash
opencrawler download --url <URL> --category <CATEGORY_FOLDER_NAME> [--storage <PATH>]
```

| 參數 | 必/選 | 說明 |
|------|-------|------|
| `--url` | 必 | 目標網頁 |
| `--category` | 必 | 分類資料夾名;不存在時自動建立(單層) |
| `--storage` | 選 | 覆寫儲存根目錄,測試時指向暫存資料夾 |

### 6.5.2 流程

```
Cli.Program.Main(args)
   │
   ▼ System.CommandLine 解析
   │
   ▼
DownloadCommand.Handler
   │
   ├─ 組 DI 容器(與 GUI 相同的 ServiceCollection)
   ├─ 若給 --storage,建臨時 AppConfig 覆寫
   ├─ DbInitializer.Init()     ← SqlSugar CodeFirst 確保表存在
   ├─ ICategoryService.EnsureByFolderNameAsync(category)
   ├─ IArticleService.DownloadAsync(url, categoryId, progress=null)
   │
   ▼ 成功
   stdout: { "articleId":..., "title":..., "folderPath":..., "imageCount":... }
   exit 0

   ▼ 失敗
   stderr: 錯誤訊息
   exit  1(參數錯) | 2(網路/HTTP) | 3(本地 IO) | 4(DB)
```

## 6.6 雙層下載策略(反爬蟲 fallback)

### 6.6.1 架構

```
              WebDownloader (協調)
                    │
         ┌──────────┴──────────┐
         ▼                     ▼
  FastHtmlFetcher       BrowserHtmlFetcher
  (HttpClient           (Playwright lazy
   + AngleSharp)         -initialized)
         │                     │
         ▼                     ▼
  ChallengeDetector       (直接回 HTML)
   判斷是否遭擋
```

### 6.6.2 IHtmlFetcher 介面

```csharp
public interface IHtmlFetcher
{
    FetchMode Mode { get; }
    Task<FetchResult> FetchAsync(string url, CancellationToken ct);
}

public enum FetchMode { Fast, Browser }

public record FetchResult(
    string Html,
    string FinalUrl,           // 經過 redirect 的最終 URL
    IReadOnlyDictionary<string,string> Headers);
```

### 6.6.3 自動模式流程

```csharp
public async Task<FetchResult> FetchAutoAsync(string url, CancellationToken ct)
{
    // 1. 先試快速模式
    try {
        var result = await _fast.FetchAsync(url, ct);
        if (!_detector.LooksLikeChallenge(result))
            return result;
        _log.Info("Challenge detected, falling back to browser mode");
    } catch (HttpRequestException ex) when (IsBlockedByAntiBot(ex)) {
        _log.Info("Blocked by anti-bot, falling back to browser mode");
    }

    // 2. 瀏覽器模式
    await _browserInstaller.EnsureInstalledAsync(ct);  // 首次下載 chromium
    return await _browser.FetchAsync(url, ct);
}
```

### 6.6.4 ChallengeDetector 規則

| 條件 | 判定 |
|------|------|
| HTTP 403 / 429 / 503 | challenge |
| 回應 body < 500 bytes | challenge |
| body 含 `"Just a moment..."`、`"Enable JavaScript"`、`"Checking your browser"`、`"cf-challenge"` | challenge |
| 解析後 `<body>` 純文字 < 200 字元且無主要圖片 | challenge |
| 其他 | 正常 |

### 6.6.5 Playwright 延遲安裝

```csharp
public class PlaywrightInstaller
{
    private bool _installed = false;
    public async Task EnsureInstalledAsync(CancellationToken ct)
    {
        if (_installed) return;
        // Microsoft.Playwright.Program.Main(new[] { "install", "chromium" })
        // 進度透過 UI 顯示(首次約 150 MB)
        var exitCode = Microsoft.Playwright.Program.Main(new[] { "install", "chromium" });
        if (exitCode != 0) throw new PlaywrightInstallException();
        _installed = true;
    }
}
```

**UI 互動**:首次需要瀏覽器模式時,彈對話框:「需下載 Chromium(約 150 MB),是否繼續?」
- 同意 → 顯示進度條
- 拒絕 → 本次下載失敗;設定內有「停用瀏覽器模式」開關,拒絕後自動勾起

## 7. SQLite 實體與存取

### 7.1 檔案位置
`{AppConfig.StorageRoot}/opencrawler.db`

首次啟動時 `DbInitializer` 透過 SqlSugar **CodeFirst** 自動建表 / 補欄位,未來加欄位只需在 Entity 上補屬性即可無痛升版。

### 7.2 連線管理
- 使用 `SqlSugarScope`(執行緒安全,可註冊為 Singleton)
- 底層 provider:`DbType.Sqlite`
- 連線字串加上 `Journal Mode=Wal;Foreign Keys=True`

### 7.3 Entity 範例(SqlSugar 資料標註)


### 7.4 DbInitializer(CodeFirst)

```csharp
public class DbInitializer
{
    public static void Init(ISqlSugarClient db)
    {
        db.DbMaintenance.CreateDatabase();         // SQLite 檔不存在則建立
        db.CodeFirst.InitTables(
            typeof(Category),
            typeof(Article),
            typeof(Notebook),
            typeof(Source),
            typeof(Summary));                       // 自動建表/補欄位
    }
}
```


> **CASCADE 實作**:`IArticleService.DeleteAsync` 在 transaction 內先 `db.Deleteable<ArticleNote>().Where(n => n.ArticleId == id)`,再刪 article 本體,避免依賴 SQLite DDL CASCADE(SqlSugar CodeFirst 不產生)。

## 11.5 在地化(i18n)設計

v1 支援 **繁體中文(zh-TW)** 與 **英文(en)**。以下列明實作方式,確保字串集中管理、可即時切換。

### 11.5.1 資源檔布局

```
src/OpenCrawler.Core/Resources/
    Strings.resx            # 預設(英文基準)
    Strings.zh-TW.resx      # 繁體中文
```

`Strings.resx` 的 `Access Modifier` 設 `Public`,由 `resx` 自動產生強型別 `Strings` 類別,程式碼可 `Strings.DownloadButton`,ViewModel 也可注入 `IStringLocalizer<Strings>` 取得。

### 11.5.2 XAML 標記擴充

自訂 `TrExtension : MarkupExtension`,回傳 `IObservable<string>`,文化變更時自動更新綁定:

```xml
<Button Content="{loc:Tr Key=DownloadButton}" />
<TextBlock Text="{loc:Tr Key=Article_DownloadedAt, Fallback='Downloaded at'}" />
```

實作概念:
```csharp
public class TrExtension : MarkupExtension
{
    public string Key { get; set; } = "";
    public string? Fallback { get; set; }
    public override object ProvideValue(IServiceProvider sp)
        => new Binding
        {
            Source = LocalizationManager.Instance,
            Path   = $"[{Key}]",           // indexer 以 Key 取值
            Mode   = BindingMode.OneWay
        };
}

public class LocalizationManager : INotifyPropertyChanged
{
    public static LocalizationManager Instance { get; } = new();
    public string this[string key] => Strings.ResourceManager.GetString(key, _culture) ?? key;
    public void SetCulture(CultureInfo c) {
        _culture = c;
        CultureInfo.CurrentUICulture = c;
        PropertyChanged?.Invoke(this, new(nameof(Item)));
        PropertyChanged?.Invoke(this, new("Item[]"));   // 觸發所有 indexer binding 重算
    }
}
```

### 11.5.3 切換流程

```
設定頁語系下拉(繁中 / English)
   │
   ▼
SettingsViewModel.LanguageChangedCommand
   ├─ IConfigService.SaveAsync( cfg with { UiLanguage = "zh-TW" | "en" } )
   └─ LocalizationManager.Instance.SetCulture(CultureInfo.GetCultureInfo(lang))
        │
        ▼ 所有 {loc:Tr} binding 重算
        所有 UI 立即更新,不需重啟
```

### 11.5.4 啟動時決定語系

```csharp
var lang = config.UiLanguage;
if (string.IsNullOrEmpty(lang))
    lang = CultureInfo.CurrentUICulture.Name.StartsWith("zh") ? "zh-TW" : "en";
LocalizationManager.Instance.SetCulture(CultureInfo.GetCultureInfo(lang));
```

### 11.5.5 字串覆蓋範圍

| 範圍 | 做法 |
|------|------|
| 所有 View(XAML) | 全走 `{loc:Tr}`,禁止寫死中英文 |
| ViewModel 錯誤訊息 | 注入 `IStringLocalizer<Strings>` |
| 例外訊息(給 log) | 保留英文,不翻譯 |
| 日期時間顯示 | `ToString(CultureInfo.CurrentUICulture)` |
| 檔案 / 目錄名 | 不翻譯(以使用者輸入為準) |

### 11.5.6 漏譯防護

CI 時跑一個簡易檢查:`Strings.zh-TW.resx` 若缺少 `Strings.resx` 的任何 key,測試失敗。

### 11.6.2 Service 介面(多 note)

### 11.6.3 自動儲存(debounce + 立即)

```csharp
public class ArticleNoteService : IArticleNoteService
{
    private readonly ConcurrentDictionary<long, CancellationTokenSource> _debounces = new();
    private readonly SemaphoreSlim _gate = new(1, 1);

    public Task ScheduleSaveAsync(long noteId, string content)
    {
        var cts = new CancellationTokenSource();
        _debounces.AddOrUpdate(noteId, cts, (_, old) => { old.Cancel(); return cts; });
        return Task.Run(async () =>
        {
            try {
                await Task.Delay(TimeSpan.FromSeconds(2), cts.Token);
                await SaveNowAsync(noteId, content, cts.Token);
            } catch (TaskCanceledException) { /* 被下一次輸入打斷 */ }
        }, cts.Token);
    }

    public async Task SaveNowAsync(long noteId, string content, CancellationToken ct)
    {
        await _gate.WaitAsync(ct);
        try {
            var now = DateTime.UtcNow;
            var bytes = System.Text.Encoding.UTF8.GetByteCount(content);
            await _db.Updateable<ArticleNote>()
                .SetColumns(n => new ArticleNote {
                    Content   = content,
                    UpdatedAt = now,
                    CharCount = content.Length,
                    ByteSize  = bytes
                })
                .Where(n => n.Id == noteId)
                .ExecuteCommandAsync();
        } finally { _gate.Release(); }
    }
}
```

- **2 秒 debounce** 每個 noteId 獨立計時
- **失去焦點 / 切換 note / 關閉分頁 / App 關閉**:呼叫 `SaveNowAsync` 立即寫入
- ViewModel 訂閱 `ByteSize` 變化 → 觸發容量警示狀態(80%/95%/100%)

### 11.6.4 容量警示邏輯

```csharp
public enum NoteSizeLevel { Ok, Warning, Danger, Over }
public static class NoteSizeEvaluator
{
    public const long SoftLimit = 2L * 1024 * 1024;   // 2 MB
    public static NoteSizeLevel Evaluate(long bytes) => (bytes * 100 / SoftLimit) switch
    {
        < 80   => NoteSizeLevel.Ok,
        < 95   => NoteSizeLevel.Warning,
        < 100  => NoteSizeLevel.Danger,
        _      => NoteSizeLevel.Over
    };
}
```

UI 對每個等級的行為:

| 等級 | 狀態列顯示 | 額外互動 |
|------|-----------|---------|
| Ok | 灰色 `1.2 MB / 2 MB` | 無 |
| Warning(≥ 80%) | 黃色 + 小 icon | Tooltip 提示「建議另起 note」 |
| Danger(≥ 95%) | 紅字 | 一次性 toast(同一 note 僅提醒一次,重啟 App 才重置) |
| Over(≥ 100%) | 紅底白字持續顯示 | 每次自動儲存完後顯示 toast「超過建議上限,效能可能下降」 |

**不強制阻擋輸入**,只提示;使用者知情選擇。

### 11.6.5 拖曳圖片

```
DragDrop:
  1. 複製到 {articleFolder}/assets/notes/<guid>.<ext>
```

### 11.6.6 效能考量

- AvaloniaEdit 使用 `TextDocument`,支援 MB 等級文件不卡頓
- 預覽區 throttle(100 ms),避免每次按鍵都重算 AST
- 大於 100 KB 時預覽改為手動(按「重新渲染」),避免捲動卡頓

## 17. 架構決策紀錄(ADR 摘要)

- **ADR-01**:採 Avalonia 而非 MAUI。理由:MAUI 的 Mac Catalyst 在 .NET 9 下仍有視窗管理 / 檔案對話框問題;Avalonia 原生跨平台成熟。
- **ADR-02**:採 SqlSugar 而非 EF Core 或 Dapper。理由:SqlSugar 具備 CodeFirst 自動建表/補欄位(免手寫 migration),Lambda 查詢寫法接近 EF 但啟動速度遠勝;相較 Dapper 省去手寫 SQL 的反覆工,資料模型雖只有 5 表但未來可能擴充。
- **ADR-03**:v1 瀏覽採系統瀏覽器(對應 D1)。理由:WebView2/WebKit 依賴平台環境,故障難 debug,先讓 v1 穩定可用;v1.1 再加內嵌 WebView。
- **ADR-04**:NotebookLM 文字摘要用 Gemini API 補足。理由:NotebookLM Enterprise v1alpha 未提供純文字 API,改用同一個 GCP 專案的 Gemini 達成相同效果。
- **ADR-05**:使用者資料完全獨立於 App 安裝目錄。理由:可攜帶、備份、搬機器。
- **ADR-06**:下載階段剝離 JavaScript(對應 D2)。理由:安全防線 + 本地開啟後 JS 大多失效,保留反而造成錯誤訊息干擾閱讀。
- **ADR-07**:僅下載 `<img>` 與 CSS 資產(對應 D3)。理由:聚焦「閱讀性」內容,降低頻寬與儲存;影音資源在 v2 再評估。
- **ADR-08**:每次下載為獨立快照(對應 D5)。理由:避免覆寫破壞既有 NotebookLM source 對應;保留歷史版本反而更貼近「封存網頁」訴求。
- **ADR-09**:UI i18n 採 `.resx` + 自訂 `{loc:Tr}` MarkupExtension。理由:標準 .NET 在地化 + Avalonia 原生支援 binding,比第三方 library 依賴少;`LocalizationManager` 單例觸發 indexer 通知,切語系即時生效不重啟。
- **ADR-10**:文章註解採 1:N(可多份 note)、單份軟上限 2 MB、接近時僅提示不阻擋。理由:使用者可能累積很長的研究筆記;強制阻擋會破壞使用流,設計哲學為「告知並尊重使用者選擇」,同時提供「新增 note」作為軟分割方案;2 MB 是 SQLite TEXT 欄位能舒適處理且 AvaloniaEdit 渲染不卡頓的界限。
- **ADR-11**:Markdown 編輯用 AvaloniaEdit + Markdown.Avalonia 雙元件,而非單一 WebView。理由:AvaloniaEdit 原生支援大型文件捲動 + 語法上色,效能優於 WebView 載入 Monaco/CodeMirror;Markdown.Avalonia 渲染足夠支援 GFM 常見語法,避免再引入 WebView 的平台依賴。
- **ADR-12**:反爬蟲以 Playwright 瀏覽器模式作為 fallback,而非純 HttpClient 加 header 仿冒或 Puppeteer-sharp。理由:Playwright 是微軟維護的官方 .NET 套件,Chromium binary 管理與 auto-install 現成可用;瀏覽器模式可同時解 Cloudflare 簡易挑戰、JS SPA 渲染兩個問題。Binary 延遲下載避免所有使用者都要為偶爾用到的功能付 150 MB 代價。
